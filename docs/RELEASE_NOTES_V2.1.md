# 🎉 视频分割拼接工具 V2.1 发布说明

**发布日期**: 2026-01-04
**版本类型**: 稳定性和用户体验重大改进

---

## 🌟 本次更新重点

V2.1 是一个以**稳定性和用户体验**为核心的重要更新，修复了多个严重问题，显著提升了程序的可靠性。

### 为什么要升级？

- ❌ **V2.0问题**: 临时文件持续累积，长期使用可能占用GB级磁盘空间
- ✅ **V2.1修复**: 自动清理，保持磁盘整洁

- ❌ **V2.0问题**: 错误提示不明确，用户不知道如何解决
- ✅ **V2.1修复**: 智能错误诊断，自动提供解决方案

- ❌ **V2.0问题**: 程序意外关闭时可能导致输出视频损坏
- ✅ **V2.1修复**: 关闭前确认，确保处理完整

---

## ✅ 主要改进

### 1. 日志系统 📝
**您的收益**: 出现问题时可以查看详细日志，快速定位原因

- 自动记录所有操作和错误信息
- 日志文件按日期分割: `logs/video_tool_YYYYMMDD.log`
- 自动清理7天前的旧日志
- 便于问题诊断和技术支持

**使用场景**:
处理失败时，可以查看日志了解具体原因，而不是只看到"处理失败"。

---

### 2. 临时文件自动清理 🧹
**您的收益**: 不再担心磁盘空间被临时文件占满

- 程序启动时自动清理3天前的临时文件
- 程序退出时清理当前会话的临时文件
- 显示临时目录大小统计

**实际效果**:
- V2.0: 使用一个月后临时目录可能有几GB
- V2.1: 临时目录始终保持在几MB以内

---

### 3. 智能错误诊断 🔍
**您的收益**: 遇到错误时知道如何解决，不再迷茫

**错误消息对比**:

**V2.0 (不友好)**:
```
错误: FFmpeg处理失败，请检查视频文件格式
```

**V2.1 (友好)**:
```
错误: 您选择了音频选项，但视频没有音频轨道

建议的解决方法:
1. 在音频设置中选择'静音'选项
2. 添加自定义音频文件
3. 使用带有音频的视频
4. 如果两个视频都没有音频，请选择'静音'
```

**支持的错误类型**:
- ✅ 文件不存在/路径问题
- ✅ 音频配置错误
- ✅ 视频文件损坏
- ✅ 权限不足
- ✅ 磁盘空间不足
- ✅ 编码失败
- ...等更多

---

### 4. 输入验证 ✔️
**您的收益**: 点击"开始处理"前就发现问题，节省时间

**验证项目**:
- ✅ 视频文件是否存在
- ✅ 视频文件是否可读
- ✅ 视频格式是否支持
- ✅ 视频文件是否完整
- ✅ 输出目录是否有写入权限

**实际效果**:
- V2.0: 处理到一半发现文件问题，前面的视频白处理了
- V2.1: 开始前就发现并提示，立即修正

---

### 5. 程序退出保护 🛡️
**您的收益**: 不会因误关闭窗口导致视频文件损坏

**新增功能**:
- 处理中关闭程序会弹出确认对话框
- 明确警告可能导致文件损坏
- 用户可选择等待或强制退出
- 记录退出原因到日志

**示例对话框**:
```
正在处理视频，强制退出可能导致输出文件损坏。

确定要退出吗？
[是(Y)]  [否(N)]
```

---

### 6. 异常处理规范化 🔧
**您的收益**: 程序更稳定，不会出现无响应或奇怪的错误

**技术改进**:
- 修复6处危险的裸异常捕获
- 所有错误都有明确的处理逻辑
- 错误信息详细记录到日志
- 可以用Ctrl+C中断程序（以前不行）

## 🔄 升级指南

### 从V2.0升级到V2.1

**方法1: 直接替换（推荐）**
1. 下载V2.1安装包
2. 解压到新目录
3. 删除旧的V2.0目录（可选）
4. 使用新版本

**方法2: 覆盖安装**
1. 备份V2.0的配置（如有）
2. 解压V2.1到V2.0目录，覆盖文件
3. 首次运行会自动清理旧临时文件

**数据迁移**: 无需迁移，V2.1完全兼容V2.0的操作方式

## 🐛 已知问题

1. **拼接后视频封面预览性能差**
   - 原因: 每次预览需生成15秒拼接视频
   - 临时方案: 可移除此选项或耐心等待
   - 计划: V2.2改为单帧提取

2. **进度条不显示实时进度**
   - 原因: FFmpeg进度未实时解析
   - 影响: 进度条不准确，但处理仍在进行
   - 计划: V2.2实现实时进度显示

## 💡 使用建议

### 查看日志
如果遇到问题，查看日志文件可以了解详情：

**Windows**:
```
程序目录\logs\video_tool_20260104.log
```

**macOS**:
```
程序目录/logs/video_tool_20260104.log
```

### 手动清理临时文件
如果需要立即释放磁盘空间：

**Windows**:
```
C:\Users\你的用户名\AppData\Local\Temp\video_pin
```

**macOS**:
```
/tmp/video_pin
```

可以安全删除该目录下的所有文件（程序会自动重建）。
